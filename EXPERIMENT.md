# Experiment Log

Detailed results and analysis for all RAG variants evaluated on the Vietnamese document corpus  
(Đoàn-Hội student documents — **50 Q&A pairs**, generated by RAGAS).

> Full reproduction commands: [Reproduce](#reproduce) · Summary chart: [Comparison](#comparison)

---

## Setup

| Parameter | Value |
|-----------|-------|
| LLM (Generator) | GPT-4o-mini (temperature=0, seed=42) |
| LLM (RAGAS Judge) | GPT-4o-mini ⚠️ *same model — see note below* |
| Embedding | BAAI/bge-m3 (GPU FP16) |
| Reranker | Qwen3-Reranker-0.6B (GPU FP16) |
| Chunk size | 1024 chars, overlap 128 |
| Top-K retrieve | 3 (baseline) / 10 → 3 (with reranker) |
| Testset size | 50 Q&A pairs |
| Evaluation framework | RAGAS 0.2.15 — 4 metrics |

> ⚠️ **Judge bias note:** GPT-4o-mini serves as both the answer generator and the RAGAS judge. This may introduce self-consistency bias — scores reflect relative improvement between variants rather than absolute quality.

## Key Findings

**Best per metric:**
- 🥇 **Faithfulness**: v4 Cross-Encoder Reranking — 0.9112 (+6.6%)
- 🥇 **Answer Relevancy**: v2 Query Decomposition — 0.7134 (+23.9%)
- 🥇 **Context Precision**: v4 & Combo v1+v2+v4 — 0.9330 (+5.1%, tied)
- 🥇 **Context Recall**: Combo v1+v2+v4 — 0.7863 (+14.2%)

**Observations:**

- **v2 Query Decomposition** gives the largest boost to Answer Relevancy (+13.8%) by breaking complex questions into targeted sub-questions, allowing the retriever to surface more relevant context per question segment.

- **v4 Cross-Encoder Reranking** consistently improves Faithfulness (+5.6%) and Context Precision (+4.6%) by filtering out off-topic chunks before generation — the most impactful single upgrade.

- **v3 Hybrid Search (RRF)** improves Context Precision (+3.3%) but slightly decreases Context Recall (−0.7%). BM25 on Vietnamese text without dedicated tokenization (underthesea not tuned) may introduce noise for keyword-sparse queries. When combined with v1 SemanticChunk (which improves tokenization quality), the effect is positive in Full Pipeline.

- **v5 CoT+Citation hurts Faithfulness (−6.5%):** The `<think>` reasoning block prepended by Qwen3's chain-of-thought output appears to confuse the RAGAS faithfulness scorer, which compares answer sentences against retrieved context. The verbose reasoning sentences don't map cleanly to context chunks.

- **Full Pipeline (v1+v2+v3+v4)** is the most balanced combo across all 4 metrics — no single metric is sacrificed.

- **Combo v1+v2+v4** achieves the best Context Recall (+14.2%) with fewer components than Full Pipeline — dropping v3 actually helps Recall, consistent with v3's standalone recall regression.

**Recommendations:**

| Use case | Best variant | Reason |
|----------|:------------:|--------|
| Accuracy-focused (minimize hallucination) | Full Pipeline (v1+v2+v3+v4) | Most balanced; highest Faithfulness + Precision combo |
| Recall-focused (don't miss relevant info) | Combo v1+v2+v4 | Best Context Recall (+14.2%) |
| Fast deployment (minimal complexity) | v4 alone | Best Faithfulness with just 1 added component |
| Avoid | v5 standalone | Reduces Faithfulness; citation format best used inside app, not eval |

## Reproduce

```bash
# === Phase 2: Baseline ===
python phase2_baseline/run_pipeline.py
python phase2_baseline/run_eval.py

# === Phase 3: Single-component upgrades ===
python phase3_upgrades/v1_indexing_chunking/runv1.py
python phase2_baseline/run_eval.py --input outputs/phase3_upgrades/v1_semantic_chunker.csv

python phase3_upgrades/v2_query_transformation/runv2.py
python phase2_baseline/run_eval.py --input outputs/phase3_upgrades/v2_query_decomposition.csv

python phase3_upgrades/v3_retrieval_strategy/runv3.py
python phase2_baseline/run_eval.py --input outputs/phase3_upgrades/v3_hybrid_search.csv

# v4: 2-step due to VRAM constraint (BGE-M3 + Qwen3 cannot run simultaneously on 6 GB GPU)
python phase3_upgrades/v4_reranking/runv4_index_retrieve.py   # Step 1: BGE-M3
python phase3_upgrades/v4_reranking/runv4_rerank_generate.py  # Step 2: Qwen3
python phase2_baseline/run_eval.py --input outputs/phase3_upgrades/v4_reranking.csv

python phase3_upgrades/v5_generation_advanced/runv5.py
python phase2_baseline/run_eval.py --input outputs/phase3_upgrades/v5_cot_citation.csv

# === Combo: Full Pipeline (v1+v2+v3+v4) ===
python phase3_upgrades/full_pipeline/run_step1_chunk_index_retrieve.py  # Step 1: BGE-M3
python phase3_upgrades/full_pipeline/run_step2_rerank_generate.py        # Step 2: Qwen3
python phase2_baseline/run_eval.py --input outputs/full_pipeline/full_pipeline_results.csv

# === Combo: Semantic Decompose Rerank (v1+v2+v4) — Best Recall ===
python phase3_upgrades/semantic_decompose_rerank/run_step1_chunk_retrieve.py  # Step 1: BGE-M3
python phase3_upgrades/semantic_decompose_rerank/run_step2_rerank_generate.py  # Step 2: Qwen3
python phase2_baseline/run_eval.py --input outputs/semantic_decompose_rerank/semantic_decompose_rerank_results.csv

# === Generate comparison report ===
python compare_results.py
```
